ロジットバイナリクロスエントロピーを使う理由

2020年02月18日 · データサイエンス

ニューラルネットで二値分類を行う場合、本やウェブ上の説明ではよく 「出力層の活性化関数はシグモイド関数を使って事後確率をモデル化し、バイナリクロスエントロピーを使って損失を計算する」 と書いてあります。

しかしながら例えば TensorFlow の DCGAN のチュートリアルを読むと、 フェイク画像と真の画像の二値分類を行う Discriminator の出力層では活性化関数を適用しておらず(恒等関数を適用しているとも言えますが)、 損失を計算するところでは from_logits=True としてロジットバイナリクロスエントロピー (シグモイド関数とバイナリクロスエントロピーの合成関数) を適用しているのですが、この理由がよくわかっていませんでした。

Numberical stability が理由だとは色々なところ ( Why is there no support for directly computing cross entropy? #2462, Tensorflow: scaled logits with cross entropy など) に書いてあるのですが、結局どういうことなのか？というのが気になったので調べました。

なお、ここまで TensorFlow の例を出して説明してきましたが、私が普段使っているからという理由で ここからは Julia の機械学習フレームワーク Flux.jl や 自動微分フレームワーク Zygote.jl で説明を実施します。

まず、シグモイド関数、バイナリクロスエントロピー、ロジットバイナリクロスエントロピーのそれぞれの定義を確認していきます。

ご存知の通り、数学的な関数としては、シグモイド関数 \( \sigma : \mathbb{R} \rightarrow (0, 1) \) は \( \sigma(x) = \frac{1}{1 + e^{-x}} \), バイナリクロスエントロピー \( \text{bce}(\hat{y}, y) : [0, 1] \times \{ 0, 1 \} \rightarrow \mathbb{R} \) は \( \text{bce}(\hat{y}, y) = -y \log \hat{y} - (1-y) \log (1-\hat{y})\) で定義されています。
